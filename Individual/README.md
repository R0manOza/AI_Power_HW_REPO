# Individual — Homework 1 (Lab 1)

This folder contains the implementation for `Task 3–5` from `homework1/Lab-1/homework/hw1-individual.md`:

- Load `GEMINI_API_KEY` from `.env` (via `python-dotenv`)
- Call two Gemini models
- Log input/output/total tokens, latency, and paid-tier cost equivalent
- Write a short 5-sentence reflection (below)

## How to run

1. Ensure your top-level `.env` contains:
   - `GEMINI_API_KEY=...`
2. Install dependencies (once):
   - `pip install -r Individual/requirements.txt`
3. Run:
   - `python Individual/hw1_compare_models.py`
4. Update this README automatically:
   - `python Individual/update_readme_from_results.py`

## Reflection (5 sentences)

1. In my run, `gemini-3.1-flash-lite-preview` produced more output tokens than `gemini-3-flash-preview`, so its paid-tier reference cost was higher even though both calls used the same prompt.
2. The call to `gemini-3-flash-preview` was slower by about 3085 ms (based on measured latency), which surprised me because the models are in the same Flash-family tier.
3. This difference showed up mainly in generation length: the prompt is identical, but `gemini-3.1-flash-lite-preview` used more output tokens, confirming that output tokens dominate cost and not just input size.
4. Both responses followed the requested checklist format, but `gemini-3.1-flash-lite-preview` generated a longer checklist (higher output-token count), so it felt more step-by-step while staying beginner-friendly.
5. If I wanted to reduce cost, I would tighten the prompt to request a shorter checklist and fewer clarifying questions, then re-run and verify that `gemini-3.1-flash-lite-preview` produces fewer output tokens.
